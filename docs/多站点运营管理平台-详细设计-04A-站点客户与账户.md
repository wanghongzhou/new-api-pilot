## 15. 站点管理

### 15.1 状态

站点使用五组独立状态：

| 状态 | 值 |
|---|---|
| management_status | active/disabled |
| online_status | unknown/online/offline |
| auth_status | unauthorized/authorized/expired |
| statistics_status | pending_config/backfilling/ready/partial/error/paused |
| health_status | ok/warning/critical/unavailable |

### 15.2 新建流程

1. 输入名称、base_url、备注；
2. 校验 URL 和唯一性；
3. 保存站点，默认未授权；
4. 选择“录入已有 Token（推荐）”或“登录并重新生成 Token（可选）”完成授权；
5. 使用 `/api/user/self` 校验 root 身份；
6. 精确读取候选 root，并通过完整用户快照校验其为最小正 ID、created_at 不晚于任一用户；
7. 将已验证 Token、root_created_at 和自动计算的 statistics_start_at 原子保存；
8. 检查必要接口和 enable_data_export，再同步通道；步骤 6 的完整快照直接作为首次用户同步结果；
9. 自动启动全部历史回填，回填期间允许查看部分数据；
10. 完成后进入 ready。身份或 root 校验失败时不保存 Token；后续能力检查失败时保留已验证授权和不可变起点，站点停在 pending_config/error 并允许从对应步骤重试。密码和 Cookie 始终不保存。

### 15.3 停用与恢复

- 已有历史的站点只能停用，不能删除；
- 停用后暂停探活、资源、RPM/TPM 和用量采集；
- 历史数据永久保留；
- 全局统计标记存在暂停站点；
- 恢复后从最后游标补齐全部可用历史；
- 补齐后重算站点及全局统计；
- 永久下线可设置 statistics_end_at，之后不视为数据缺失。
- statistics_end_at 仅允许在站点已停用时设置，必须对齐北京时间整点，并位于 `[coalesce(last_complete_hour+3600, statistics_start_at), disabled_at]`；清除后仍需显式恢复站点并补齐。

### 15.4 当前指标

- 在线状态：60 秒；
- 当前 RPM/TPM：60 秒，仅保存在 site 当前字段；
- 实例当前状态：60 秒；
- 业务统计：截至最近完整小时；
- 所有页面显示数据更新时间和完整状态。

站点运营概览口径：

- 站点列表固定比较最近 24 小时，显示请求数、额度、Token 数、平均 RPM、平均 TPM、成功率、平均延迟、吞吐量、实例与资源健康；不展示模型明细。卡片和表格使用同一组字段和时间口径。
- 用量来自本地完整小时事实：请求、额度和 Token 在 `site_stat_hourly` 求和；活跃用户必须在 `usage_fact_hourly JOIN collection_window(status=complete)` 的范围内按 `remote_user_id` 去重，不能相加每小时活跃数。无事实但已完成的窗口数值为 0。
- 站点详情概览支持 24 小时、7 天、30 天和自定义已闭合时间范围。本地用量按所选范围汇总；平均 RPM/TPM 为请求数/Token 数除以该范围内有效覆盖分钟数，不能平均各小时比率。
- 性能健康独立于用量统计。站点详情在用户选择范围时实时透传 `hours=1..720` 到上游 `/api/perf-metrics/summary`；`hours=24` 的成功结果同时写入按 `site_id + config_version` 隔离的进程内缓存。站点列表只读取该缓存，缓存缺失时本次先返回 `unavailable` 并异步刷新，不阻塞列表等待上游；成功 TTL 1 小时，失败 TTL 5 分钟，进程重启后丢失。性能数据不落 MySQL 或 Redis，不在本地累加滚动汇总。
- 性能健康显示成功率、平均延迟、吞吐量和上游返回的模型性能列表。总体成功率和均值由后端按各模型 request_count 加权；不得把各次滚动快照相加、擅自截断模型或将各模型/各小时百分比直接平均。
- 实时、业务统计、性能健康分别显示 `realtime.updated_at`、`usage.as_of`、`performance.sampled_at`。卡片和表格不得以同一个“更新时间”混用。

站点列表顶部工具栏：

- 搜索：站点名称、API 地址；
- 筛选：management_status、online_status、auth_status、statistics_status、health_status；
- 卡片/表格视图切换；

卡片规则：

- 在线、授权、统计、资源健康使用独立徽标；
- 当前 RPM/TPM 超过两个周期未更新时显示“已过期”；
- 所有用户界面将 `quota` 称为“额度”；
- 列表用量固定读取最近 24 小时完整统计窗口，不使用“今日”或最近一条历史 daily 冒充该范围；
- 金额由前端按站点当前汇率计算；
- backfilling/partial 时显示进度和完整率；
- 数值字段无样本时显示 0（百分比显示 0.0%），字符串和时间字段显示 `-`；`data_status`、采样/截止时间仍必须保留，避免将“零使用量”和“尚未采样”混淆；
- disabled 站点降低饱和度并显示停用时间。

新建/编辑站点 Drawer：

- 站点名称，必填；
- API 地址，必填，自动去尾斜杠并校验唯一；
- 备注；
- 编辑 API 地址先调用候选地址预检接口执行不携带 Token 的公开探活，展示规范化地址、新旧 origin/path、system_name 和 version；admin 确认同一逻辑站点后提交带时效预检凭证的 PUT。scheme/host/port 变化时立即标记 unauthorized 并引导重新授权，仅同主机路径变化才允许继续校验原 Token；
- 新建成功后引导授权。

授权弹窗：

- 默认模式：root_user_id、已有 Access Token；说明不会修改远端 Token；
- 可选模式：root 用户名、root 密码、`confirm_token_rotation=true` 二次确认；
- 可选模式必须明确提示密码仅用于换取 Access Token、不保存密码，并说明会覆盖远端原 Access Token；
- 授权成功后显示能力检查结果；
- data_export 未开启时显示“统计待配置”；
- 上游登录明确拒绝、授权过期/权限错误、root 角色或首用户证明错误、接口不兼容分别给出明确提示；登录拒绝只显示“请检查用户名或密码”，不得回显或保存上游 message、密码与 Cookie。

站点详情只读展示 root 注册时间、statistics_start_at 和固定来源 `root_created_at`。产品不提供修改起点入口；重新授权时必须重新完成首用户证明，新读取的 root.created_at 与已存 root_created_at 不一致或 root 已不再是快照中的首用户，则显示站点身份异常并停止采集。

disabled（含设置 statistics_end_at 的永久下线）站点允许 authorize/reauthorize/recheck-capabilities，以便先修复 expired 或配置问题再执行 enable；这些动作只更新授权、能力结果和 config_version，站点仍显示 paused，不创建回填任务。身份/能力失败显示在授权与能力区，management_status/statistics_status 仍保持 disabled/paused；只有清除 statistics_end_at 并显式 enable 后才进入 backfilling。

站点详情页：

- 概览 Tab：基本信息和五组状态；“运营概览”展示所选范围的请求、额度、Token、平均 RPM/TPM 与业务数据截至时间；“性能健康”展示成功率、平均延迟、吞吐量、模型成功率与性能采样时间；“资源与实例”展示实例、CPU、内存、磁盘与资源采样时间。模型请求、额度和 Token 明细仅在统计 Tab 展示；
- 实例 Tab：跳转或嵌入实例状态页面；
- 统计 Tab：跳转或嵌入站点详细统计；
- 采集 Tab：最近探活、小时采集、历史回填、次日校验和每周校验记录；
- 回填中展示已完成范围、待处理范围、失败窗口和重试按钮。
- 点击任务通过 `/api/collection-runs/:id/windows` 分页查看该 run 的执行窗口；重试会创建新 run 及新的 run-window，只覆盖原失败/缺失小时范围，不改写旧 run 历史。

实例状态页：

- 顶部汇总：实例总数、在线实例数、CPU/内存/磁盘最高值、health_status；
- 当前实例表：node_name、hostname、master、在线状态、运行版本、CPU、内存、磁盘、started_at、last_seen_at；
- 趋势支持分钟、小时、日粒度；
- 分钟趋势只允许查询 90 天留存范围内的数据；
- 可选择全部实例或单个实例；
- CPU/内存图表支持最大值、平均值切换；磁盘支持最大值、期末值切换，不能把期末值标成平均值；
- 数据缺口显示断点，不使用 0 连线。

- 时间范围和小时/日/月/年粒度；

- Excel/CSV 导出；

## 16. 账户管理

### 16.1 定位

“添加账户”指在平台新增一个需要统计的纳管账户，不是在远端 new-api 创建用户。

### 16.2 添加流程

1. 选择 active 且已授权的站点；
2. 调用站点用户搜索接口；
3. 选择远端用户；
4. 选择 using 状态客户；
5. 校验 site_id + remote_user_id 唯一，且远端 created_at 在 (0,now]；
6. 使用精确读取的最新远端属性创建 account，固化 remote_created_at，设置 remote_missing_count=0、last_remote_seen_at=当前时间；
7. 从基础事实生成全部历史账户统计；
8. 重算客户历史统计；
9. 显示回填状态。

### 16.3 不变量

- 一个站点远端用户只能纳管一次；
- 一个账户只能属于一个客户；
- site_id、remote_user_id、customer_id 永久不可修改；
- remote_created_at 创建后不可修改，账户历史从其北京时间整点开始；
- 不提供客户绑定中间表；
- 不管理远端用户 API Token；
- 不在远端创建、充值、调额或禁用用户。

添加纳管账户弹窗：

~~~text
选择站点
   ↓
输入用户名或用户 ID 搜索远端用户
   ↓
选择远端用户并展示当前属性
   ↓
选择客户
   ↓

   ↓

~~~

- 站点必须 active、已授权且 data_export 已开启；
- 客户必须为 using；communicating、signing 和 disabled 客户不能添加纳管账户；
- 已被纳管的 site_id + remote_user_id 置灰并提示所属客户；
- 提交前明确提示三个绑定字段创建后不可修改；
- 创建后立即进入账户详情并展示回填进度。

账户详情：

- 基本信息区：站点、客户、远端用户 ID、远端注册时间、用户名、显示名、分组；
- 当前状态区：远端状态、remote_missing、当前 quota、累计已用、最近同步时间；
- 纳管状态区：active/archived、暂停时间、恢复回填状态；
- 今日摘要：请求数、quota、金额、Token、数据截至时间；
- 统计区：小时/日/月/年趋势、表格和图形切换；模型和通道统计进入独立的全站统计页面，不把账户口径混入模型/通道物化表；
- 完整性区：以 remote_created_at 整点为账户统计起点，展示缺失范围、回填状态和最近校验时间。

账户操作：

- 编辑备注；
- 刷新远端属性；
- 归档；

## 17. 客户管理

### 17.1 状态

| 状态 | 含义 |
|---|---|
| communicating | 沟通中 |
| signing | 签约中 |
| using | 使用中 |
| disabled | 已停用 |

### 17.2 关系

- customer 与 account 为一对多；
- account.customer_id 为外键；
- 不建立 customer_account；
- 客户归属创建后不可修改。

### 17.3 停用与恢复

- disabled 时暂停客户及其账户汇总；
- 基础事实和站点、全局、模型、通道统计继续生成；
- 恢复 using 后补齐全部停用期间；identity_mismatch 账户保持暂停并在页面单独提示；
- enable 回填期间继续显示暂停前历史、恢复区间 paused 状态、已完成范围和进度；恢复区间不会逐桶开放，全部重建成功后原子开放，避免客户与所属账户在中间状态口径不一致。

# P0-C 全站用户库存与账户交集

库存覆盖站点全部远端用户，不限于已纳管 account。account 仍是客户归属和六级统计边界；库存只通过 `(site_id, remote_user_id)` 与 account 关联，不自动创建客户或 account。完整快照事务先核对库存 identity，再更新库存、小时汇总和已纳管账户；created_at 冲突使库存与 account 同时进入 identity_mismatch，禁止把新身份额度归给旧账户。

提供全局与站点库存列表/统计，筛选 role/status/group/remote_state/用户名/最后登录/余额区间。余额定义为 quota-used_quota，可为负数；增长按 remote_created_at，活跃按 last_login_at 落入查询时间范围，不从 request_count 增量猜测登录活跃。
