### 23.1 权限表现

- admin 展示全部按钮；
- viewer 隐藏所有写操作；
- viewer 保留查看、筛选、下钻和导出；
- 后端权限校验为最终依据。

### 23.2 数据状态表现

- complete：正常展示；
- pending/backfilling：显示进度和部分数据提示；
- missing：显示缺失，不补 0；
- unavailable：显示上游不可恢复；
- paused：显示暂停原因；
- 所有页面展示数据更新时间。

### 23.3 国际化

前端使用 i18next 和 react-i18next，产品语言固定为简体中文 `zh-CN`。所有用户可见文案使用翻译键，不在组件中硬编码；本期不提供语言检测、语言切换或其他 locale 资源。

### 23.4 登录与强制改密页面

- /sign-in：用户名、密码、登录按钮；
- bootstrap admin 使用环境变量或一次性密码首次登录后跳转 /change-password；
- must_change_password=true 时不能进入其他路由；
- 已登录用户可从 Header 进入 /change-password 自助修改密码；
- 修改完成后进入 Dashboard；
- 用户名不存在、密码错误和禁用用户提交错误密码只提示用户名或密码错误；凭据正确但账户禁用时明确提示账户已禁用。
- 原密码错误显示在修改密码表单的当前密码字段。
- 登录密码只校验非空；创建用户、重置密码和修改密码校验至少 8 个 Unicode 字符，并用 TextEncoder 校验 UTF-8 不超过 72 字节。

### 23.5 平台用户页面

路由：/settings/users

- 用户名、显示名、角色、状态、最近登录时间；
- admin 可新增 admin/viewer；
- admin 可编辑角色、启用、禁用和重置密码；
- 重置密码后 must_change_password=true；
- 最后一个可用 admin 的禁用按钮置灰；
- viewer 可以查看用户列表，但没有写操作。

### 23.6 系统配置页面

路由：/settings/system

- 固定 60 秒采集周期的只读说明，以及可编辑的小时采集延迟；
- 并发上限；
- 手动补采单次最大天数；
- 分钟数据留存天数；
- 导出文件有效期、单用户/全局活跃上限、最大文件和磁盘余量；
- PUBLIC_ORIGIN 只读展示，以及钉钉 Webhook、签名密钥和测试按钮；
- 汇率兜底值；
- 敏感值只显示掩码，不回显明文；
- 敏感项提供“替换”“显式清除”两个独立动作；清除必须二次确认并发送 clear=true，空输入表示不修改；
- decrypt_error 时显示稳定错误码和重新配置动作，不展示密文或后端解密详情；测试消息只使用已保存配置，未启用/未配置/解密失败时按钮禁用并说明原因；
- 页面展示 H+15 SLO eligibility；`h15_slo_reason_codes` 仅作为机器标志，不直接当作 MessageRef 渲染。延迟标志使用 `{value: usage_delay_minutes, threshold: 5}`，并发标志使用 `{value: usage_concurrency, threshold: 5}` 构造完整 i18next params 后显示生产发布阻断提示；
- 仅 admin 可修改，viewer 只读。
- 页面通过 Viewer 可读的 `GET /api/settings` 一次取得五个固定分组；`system.public_origin` 只读且不进入 PUT；普通整数始终发送 JSON number，两个字节阈值和两个 decimal 始终按十进制字符串处理，不经过 JavaScript number，decimal 输入只提交 ASCII 固定点形式；
- 保存只发送实际编辑项，不携带 updated_at/version。后端按全量行锁原子 last-write-wins；成功后以响应中的 updated_at 替换本地快照，失败时整批保留原表单值；
- Webhook/secret 空输入表示保持，替换和 clear=true 是两个显式动作；enabled=true 时两项必须均已配置且可解密。decrypt_error 项只能替换或在停用同批次中显式清除。

## 36. 路由、URL 状态和权限

### 36.1 根认证守卫

_authenticated/route.tsx 参考 new-api web/default 现有模式：

1. 本地 auth store 无用户时跳转 /sign-in?redirect=当前地址；
2. 每次浏览器会话首次进入时调用 GET /api/user/self；
3. 401 清空 auth store 并跳登录；
4. 网络错误或 5xx 不误判退出，显示全局错误并允许重试；
5. must_change_password=true 且目标不是 /change-password 时重定向；
6. must_change_password=false 时允许访问 /change-password；普通认证页面按各自权限继续访问。

登录成功保存最小用户信息和 uid；退出清除 auth store、uid、Query Cache。

uid、siteId、customerId、accountId、alertId、runId、exportId 等 BIGINT ID 在 TypeScript 中统一为 string，不允许 `Number(id)`；Unix 秒时间戳保持 number。

### 36.2 路由搜索参数

列表页 URL 必须可复制、前进后退可恢复。使用 createFileRoute.validateSearch + zod，组件用 useTableUrlState 映射。

BIGINT ID 搜索参数使用不带 JSON 引号的十进制字符串（如 `runId=9223372036854775807`）；路由解析必须保持原字符串，不得先转为 JavaScript number。历史上带 JSON 引号的深链仍需兼容读取。

## 37. TanStack Query 设计

### 37.1 Query Key

~~~text
dashboardKeys = {
  all: ['dashboard'],
  summary: () => ['dashboard','summary'],
  trend: (days) => ['dashboard','trend',{days}],
  top: (type,metric) => ['dashboard','top',{type,metric}],
  health: () => ['dashboard','health']
}

siteKeys = {
  all: ['sites'],
  lists: () => ['sites','list'],
  list: (params) => ['sites','list',stableParams(params)],
  detail: (id) => ['sites','detail',id],
  status: (id,params) => ['sites','status',id,stableParams(params)],
  instances: (id) => ['sites','instances',id],
  performance: (id,hours) => ['sites','performance',id,hours],
  runs: (id,params) => ['sites','runs',id,stableParams(params)],
  fastTaskHistory: (id,params) => ['sites','fast-task-history',id,stableParams(params)]
}

customerKeys、accountKeys、statisticsKeys、alertKeys、userKeys、
settingKeys、exportKeys 使用相同 all/list/detail 层次。
~~~

stableParams 在创建 key 前移除 undefined、排序 ID 数组和字符串数组。不能把每次 render 创建的新 Date、函数或未稳定对象放入 queryKey。

### 37.2 缓存和轮询

| 数据 | staleTime | refetchInterval |
|---|---:|---:|
| Dashboard summary/trend/top | 5 分钟 | 不自动轮询 |
| Dashboard health | 30 秒 | 60 秒 |
| 站点列表当前状态 | 30 秒 | 60 秒 |
| 站点详情当前状态 | 30 秒 | 60 秒 |
| 实例当前状态/分钟趋势 | 30 秒 | 60 秒 |
| 站点性能健康 | 30 秒 | 不自动轮询；切换 24/168/720 小时或手动刷新时请求 |
| RPM/TPM | 30 秒 | 60 秒 |
| 快速任务历史 | 5 秒 | 5 秒；离开采集区块后停止 |
| 统计详情 | 5 分钟 | 不轮询 |
| backfilling 对象和 collection run | 5 秒 | 5 秒，终态停止 |
| export job | 2 秒 | 2 秒，终态停止 |
| firing 告警 | 30 秒 | 60 秒 |
| 历史告警、用户、设置 | 30 秒 | 不轮询 |
| 站点/客户/账户选择项 | 5 分钟 | 不轮询 |

页面失去焦点不自动重跑重型统计；沿用 QueryClient refetchOnWindowFocus=false。GET 请求继续使用共享 Axios 的并发去重。

### 37.3 Mutation 失效

| Mutation | 成功后失效 |
|---|---|
| 新建/编辑/停用/恢复站点 | siteKeys.all、dashboardKeys.health、dashboardKeys.summary |
| 授权/探活/刷新站点 | 对应 site detail/list/status/runs |
| 重新检查站点能力 | 对应 site detail/list/runs；通过后轮询 backfill |
| 新建/编辑/停用客户 | customerKeys.all、accountKeys.lists、相关 statisticsKeys |
| 添加/归档/恢复账户 | accountKeys.all、customer detail/list、相关 statisticsKeys |
| 修改规则 | alertKeys.rules、dashboardKeys.health |
| 修改系统配置 | settingKeys.all；受影响的轮询配置重新读取 |
| 修改平台用户 | userKeys.all |
| 修改自己密码 | auth self；随后清除 PASSWORD_CHANGE_REQUIRED 状态 |

Mutation 成功显示 Sonner success；失败由共享 Axios/QueryClient 处理。删除和状态切换不做乐观更新，避免后端约束失败导致 UI 回滚复杂。

### 37.4 防瀑布

- Dashboard 四个接口在同一 render 中并行 useQuery；
- 站点详情基础信息、今日统计、实例和 collection run 并行；
- 统计页面的筛选选项并行加载，不等待主统计完成；
- 模型和通道筛选使用 `/api/statistics/options/*` 服务端搜索，不一次性加载全部 distinct 值；
- 只有 account 远端用户搜索依赖先选择 site_id，使用 enabled 控制；
- export mutation 不等待统计页面数据下载完成，直接提交同一筛选；
- route loader 仅预取首屏关键数据，不串行预取所有 Tab。

实例状态页一次请求 `/api/sites/:id/instances` 取得全部实例当前 CPU、内存、磁盘、sampled_at、data_status 和 effective current_status，不按实例发 N 个当前值请求。upstream_status 只放技术详情；列表徽标、health_status 和告警使用有效规则阈值计算的 online/stale/offline/unknown。历史趋势使用单个可选 node_name 查询；CPU/内存在 max/avg 间切换，磁盘在 max/期末间切换。

### 38.4 客户表格和卡片

列：名称、联系方式、状态、账户 active/total、站点数、今日请求、今日 quota/金额、活跃账户、完整状态、更新时间、操作。默认 updated_at desc。disabled 行使用共享 disabled row style，不用自定义灰色值。

新建/编辑客户是 Dialog。停用是 AlertDialog，文案明确“停用期间账户和客户统计暂停，恢复后自动补齐”。恢复按钮触发后打开 Backfill Sheet。

### 38.5 账户表格和卡片

列：用户名/remote_user_id、站点、客户、远端分组、远端状态、remote_state、managed_status、当前 quota、used_quota、今日请求、今日 quota/金额、完整状态、回填进度、更新时间、操作。默认 updated_at desc。

添加账户使用三步 Drawer：

1. SiteField：仅 active + authorized + data_export_enabled；
2. RemoteUserCombobox：500ms debounce 搜索，结果显示 ID、用户名、显示名、分组、状态、是否已纳管；
3. CustomerCombobox：仅 using；
4. 确认页显示固定绑定并提交。

步骤切换只保留一个 react-hook-form；Zod 在每步 next 时验证当前字段。提交后导航账户详情，不关闭后丢失任务反馈。

新建站点使用四步 Drawer：

1. 基础信息：名称、规范化 base_url、备注；
2. 授权：默认已有 Token，可切换登录生成 Token 并完成覆盖风险确认；
3. 能力检查：展示 root 身份、root 注册时间、自动计算且不可编辑的 statistics_start_at、版本、数据导出开关和 flow/data 校验；
4. 启动回填：展示任务范围、进度入口和失败重试。

如果 root 用户读取或 created_at 校验失败，第三步停止并展示 SITE_INCOMPATIBLE；前端不提供手工历史起点输入框。

编辑 base_url 时，离开输入框或点击“预检”调用 `/base-url-preflight`；预检结果明确展示规范化地址、origin/path 变化以及新旧站点实际返回的 system_name/version。只有预检成功且 admin 勾选“确认是同一逻辑站点”后才能提交；凭证过期或 config_version 变化时重新预检，前端不复用旧凭证。

### 38.6 告警表格

列：level、status、rule、site、target、current/threshold、first_fired_at、last_fired_at、resolved_at。默认 firing first、level desc、last_fired_at desc。Critical 使用 destructive Badge，Warning 使用 warning 语义变体，不靠颜色单独表达，必须同时显示图标和文本。

点击行打开只读 Sheet，展示事件时间线和钉钉投递结果；不显示确认、负责人、备注和关闭按钮。

`/alerts` 使用“事件 / 规则”Tabs。事件 Tab 顶部通过 `/api/alerts/summary` 展示 firing、Critical、Warning、今日恢复，列表默认使用后端复合优先级；alertId 深链打开详情。

规则 Tab 展示 rule_key、级别、metric、比较符、阈值、连续次数、global/site、继承状态和更新时间。admin 可编辑 enabled/threshold/for_times，可选择站点从全局规则创建覆盖，也可删除站点覆盖回落全局；viewer 全部只读。布尔/身份类规则的 threshold 与 compare_operator 只读，仅允许 enabled；百分比规则联动校验 Warning<Critical。保存冲突显示字段错误，不在前端悄悄调整阈值。

### 38.7 平台用户表格

列：username、display_name、role、status、must_change_password、last_login_at、created_at、操作。默认 created_at desc。最后一个启用 admin 和当前用户的禁用操作在前端置灰并解释，后端仍校验。

### 38.8 采集记录表格

持久任务表列：task_type、trigger_type、range、status、fetched_rows、written_rows、retry_count、duration、error、created_at。running/pending 每 5 秒刷新。错误列截断，点击打开 Sheet 查看完整安全错误和 request_id；不显示远端 Token。task_type 使用平台固定的 19 项目录，并按 fast、durable、hourly、usage、rebuild 分组；每项同时展示用途文案。目录如下：

- fast：`site_probe`、`realtime_stat`、`resource_snapshot`，用于站点探活、实时指标和资源快照；
- durable：`performance_sync`、`topup_sync`、`redemption_sync`、`upstream_task_sync`、`model_meta_sync`、`plan_sync`、`pricing_group_sync`、`system_task_sync`，用于各类持久化运营数据同步；
- hourly：`user_sync`、`channel_sync`、`log_sync`，按小时边界同步站点用户、渠道和日志；
- usage：`usage_hour`、`usage_backfill`、`usage_validation`，用于小时用量、历史补采和完整性校验；
- rebuild：`account_rebuild`、`customer_rebuild`，用于账户和客户统计重建。

快速任务历史只接受 fast 的三个 task_type；未知 task_type 阻止展示并显示契约错误，不以空列表掩盖服务端返回异常。

同一采集区块下方提供“快速任务历史”，调用 `/api/fast-tasks`。admin 和 viewer 均可按 `site_probe/realtime_stat/resource_snapshot` 与 `success/failed` 筛选，使用 offset/limit 展示最近执行；列为 task_type、status、started_at、finished_at、duration_ms、error、request_id。该列表明确标注为 Redis 短期历史，不与持久 collection run 合并、不承诺跨 TTL 或重启长期保留；加载失败显示独立错误，不隐藏持久任务表。

任务详情 Sheet 调用 `/api/collection-runs/:id/windows`，按 pending/running/success/failed/unavailable 执行状态筛选，分页展示 hour_ts、fact_status、attempt_count、next_retry_at、verified_at 和稳定错误码；每行是该 run 不可变归属的 collection_run_window，新 run 不覆盖旧行。“重试失败窗口”创建新 run，不能直接篡改旧 run。完成范围由 run-window 列表/计数推导，超过一页时明确显示分页而不是假装已列全。

### 38.9 详细统计 Breakdown 列

P0-A 在共享 TypeScript DTO/API client 基础上新增 group/token/node 页面与统计二级导航。路由固定为 `/statistics/groups`、`/statistics/tokens`、`/statistics/nodes`，复用现有统计页的时间范围、小时/日/月/年粒度、指标、趋势/明细、完整性、逐桶 site_breakdown 金额换算和导出闭环。StatisticsQueryParams、ExportFilters、query key 稳定化必须保留 use_groups/token_keys/node_names；联合 breakdown 增加 Group/Token/Node 变体，token_id 保持 string。

三个页面的维度选项必须分别来自 options/groups、options/tokens、options/nodes，不得从当前分页结果猜测 distinct 选项。group 使用原始 use_group，空值显示“未知分组”；token 使用 site_id:token_id 作为 token_key，token_id 包括 0 在内始终保持十进制字符串，名称为空或 0 时显示明确未知/已删除状态；node 使用原始 node_name，空值显示“未知节点”。URL 保存站点和本维度多选筛选并支持刷新、复制、前进后退；375px 使用现有响应式筛选 Sheet/卡片布局，不横向溢出。complete 零、partial、missing、unavailable、paused 的展示和无障碍语义与六个既有统计页一致。

| 页面 | 主维度列 | 附加列 |
|---|---|---|
| global | 时间桶 | 完整站点/预期站点 |
| sites | site | 状态、汇率、完整率 |
| customers | customer | 账户数、站点数、site_breakdown |
| accounts | account | site、customer、remote_user_id |
| models | model_name | site |
| channels | channel_name/channel_id | site、remote_missing |

所有页面共同显示 request_count、quota、USD、CNY、token_used、active_users、data_status、as_of。账户 active_users 按 API 的派生值展示：`"1"` 表示范围内有事实，`"0"` 表示 expected 全部完整且无事实，`null` 才显示“—”并同时展示非 complete 的 data_status；前端不得自行固定为“—”或伪造 1。

### 38.10 Dashboard

Dashboard 使用普通可滚动 SectionPageLayout，不使用固定高度表格布局：

- 今日运营：今日请求、quota/金额、Token、活跃账户，以及站点/客户/账户/实例运营实体；
- 实时吞吐：当前 RPM/TPM、完整/预期站点、过期站点和 data_status；
- 近 30 天趋势：VChart 趋势；
- 今日快速排行：站点/客户/模型/通道 Tabs；
- 站点健康、完整性与告警：站点状态、完整率、缺失范围和 firing 告警入口；

### 38.11 站点列表与详情看板

站点列表使用轻量卡片概览站点身份、状态、实例/资源、实时 RPM/TPM、当日请求/quota/Token/活跃账户和完整率，不在列表请求趋势或明细统计。

站点详情在实时概览、性能、实例和配置之外，内嵌单站数据看板。该看板复用详细统计的筛选、汇总、趋势图/数据表、数据完整性和导出任务交互，查询 `GET /api/sites/:id/stats`；详情中的筛选状态独立维护，切换范围时保留上一份合法结果直到新结果返回。原 `/sites/:siteId/stats` 路由保留为相同能力的独立深链页面。

性能健康区块提供 24 小时、7 天（168 小时）、30 天（720 小时）切换，调用 `GET /api/sites/:id/performance`。它是单站实时代理例外，不与 `/api/sites/:id/stats` 混为用量统计；显示 request_count、success_rate、avg_latency_ms、avg_tps、模型成功率、sampled_at 和 data_status。切换范围时保留上一份成功结果，错误只覆盖性能区块。站点列表中的 performance 来自服务端进程缓存，`unavailable` 时显示“暂不可用”，不得把零计数解释为已成功采样。

实例列表对 retired node 显示“已退役”和 retired_at，不再展示为当前 stale/offline，也不计入当前实例数；历史资源和已结束告警仍可下钻。节点重新出现后按新返回状态展示，不能复活旧告警文案。

区块分别处理 loading/error，不因单接口失败隐藏全页。点击卡片使用 TanStack Link 进入带好筛选参数的详细统计。

### 38.12 站点实例状态页

`/sites/:siteId/status` 展示该站点全部实例的当前 CPU、内存、磁盘、节点身份、有效状态、采样时间和 data_status；技术详情可展开查看 upstream_status 与运行时版本。页面一次读取实例当前值，不为每个实例重复请求；retired 实例显示退役状态和 retired_at，不计入当前实例数，但保留历史资源下钻。

资源趋势支持 CPU、内存、磁盘指标，按 minute/hour/day 粒度和 avg/max/last 聚合切换，并支持可选 node_name；图表之外提供可访问的数据表。health_status 使用服务端有效阈值，`online/stale/offline/unknown` 和 `ok/warning/critical/unavailable` 均以状态徽标和文字表达。实例列表、汇总和趋势分别处理加载、错误与不完整状态，不能把 unavailable 或缺失数据渲染成真实零值。

### 38.13 订阅计划目录页

交付全局 `/subscription-plans` 与强制站点 `/sites/:siteId/subscription-plans` 只读页面，主导航和站点详情均提供入口。页面明确说明“计划目录，不是订阅收入或用户订阅库存”；站点页只调用强制站点 API，不以前端过滤全局结果。全局筛选支持站点、关键词、启用状态和 normal/missing 状态，站点页隐藏站点筛选；筛选、分页和导出任务 ID 写入 URL，刷新、深链及前进后退恢复，修改筛选时页码归 1。

列表展示计划标题/副标题、站点身份、远端 ID、币种和原始价格字符串、总额度、时长、额度重置周期、启用状态、远端状态、data_status、创建/更新时间和排序值。统计展示 total/enabled/disabled/missing 以及站点 breakdown、data_status/as_of；ID、金额、额度和计数保持字符串，不由前端换算、跨站汇总或推断收入。CSV/XLSX 冻结安全筛选并排除分页，沿用 ExportTaskSheet；页面无任何写操作。

## 39. VChart 和金额展示

### 39.1 图表组件

| 组件 | VChart 类型 | 用途 |
|---|---|---|
| MetricTrendChart | line/area | 请求、quota/金额、Token、活跃数趋势 |
| RankingBarChart | horizontal bar | Top N |
| DistributionChart | donut | 站点/模型/通道占比 |
| ResourceTrendChart | multi-line | CPU、内存、磁盘 |
| CompletenessChart | gauge/progress | 完整率；优先使用普通 Progress，只有 Dashboard 才用图 |

图表通过动态 import 分包，路由首屏先渲染 Skeleton。统一从 @/lib/vchart 取得 light/dark 主题，主题切换时更新 theme prop，不在组件内复制调色板。

### 39.2 数据点

前端不重新聚合服务端趋势。每个点直接映射：

~~~text
{
  bucketStart: number
  label: string
  value: number | null
  rawValue: string | null
  dataStatus: DataStatus
  isFinal: boolean
  siteBreakdown: SiteQuotaBreakdown[]
  reason?: AnyMessageRef
}
~~~

null 点必须形成断线；complete 零值正常连到 0；partial 点用虚线段或显式空心标记，并在 tooltip 标明“仅包含已完成站点”。如果 VChart 某个系列配置会自动把 null 转 0，数据适配层必须过滤该点并显式构造断段，不能依赖默认行为。

### 39.3 Tooltip

趋势 tooltip 固定显示：

- 时间桶和北京时间；
- 原始精确值；
- 数据状态；
- is_final/as_of；
- quota 指标下的每站 quota、quota_per_unit、USD 汇率、USD、CNY；
- partial 时的 complete_site_count/expected_site_count；
- missing/unavailable 的原因，不显示 0。

原因、能力检查、任务失败和告警主文案统一使用后端 `code + params` 映射 i18next；technical_detail/error_message 只在可展开的技术详情中显示，并同时提供 request_id。组件不得把自由文本当作状态分支条件。

Top N 和占比图点击后更新 URL 筛选或进入对应详情。

### 39.4 金额公式

前端 Decimal 计算：

~~~text
amount_usd = Decimal(quota).div(Decimal(quota_per_unit))
amount_cny = amount_usd.mul(Decimal(usd_exchange_rate))
~~~

优先使用站点最近一次有效汇率；站点从未成功返回汇率时才允许使用系统兜底值，并显示“兜底汇率”。两者都缺失或非正数时金额返回 null 并显示“汇率不可用”，原始 quota 仍展示。跨站金额逐个 site_breakdown 计算后再 Decimal.sum，禁止先汇总 quota 再套任一站点汇率。

金额默认显示 2 位，tooltip 和导出显示 6 位；原始 quota 永远可查看。汇率旁显示“当前汇率”，避免被理解为历史汇率。

跨站合计中，只要任一 quota 非零的站点 rate_source=unavailable，整体 USD/CNY 返回 null 并显示“部分站点汇率不可用”，不得只把已知站点相加后伪装成完整总额；site_breakdown 仍逐站展示可计算值。

## 40. 加载、空、错误和不完整状态

### 40.1 状态优先级

页面按以下顺序决定主体状态：

1. 首次加载：Skeleton；
2. 请求失败且无旧数据：ErrorState + Retry；
3. 请求成功且无业务对象：Empty；
4. 有旧数据且后台刷新：保留内容并显示轻量 fetching；
5. backfilling/partial：Alert + 可用部分数据；
6. missing/unavailable：对应点为空和 CompletenessAlert；
7. complete：正常。

不能用全屏 Spinner 覆盖已有数据；后台刷新不清空表格。

### 40.2 Empty 文案和动作

| 页面 | Empty | admin 动作 | viewer 动作 |
|---|---|---|---|
| 站点 | 尚未添加站点 | 新建站点 | 无 |
| 客户 | 尚未创建客户 | 新建客户 | 无 |
| 账户 | 当前筛选无纳管账户 | 添加账户/重置筛选 | 重置筛选 |
| 统计 | 范围内无流量 | 调整范围 | 调整范围 |
| 告警 | 当前没有匹配告警 | 重置筛选 | 重置筛选 |
| 导出 | 尚无导出任务 | 发起导出 | 发起导出 |

“无流量”只用于后端 data_status=complete 的零结果；missing 不能复用 Empty。

### 40.3 状态颜色之外的信息

所有状态同时使用文本、图标和语义 Badge。图表线型、点形和 tooltip 一起表达 partial/missing，满足色觉无障碍。

## 41. 响应式和无障碍

### 41.1 响应式

- 小于 640px：列表使用单列卡片；筛选收进 Sheet；主操作固定在 Header 或底部 Drawer；
- 640～1024px：卡片 2 列，统计卡片 2 列；
- 大于等于 1024px：站点卡片 3 列，Dashboard 12 列网格；
- 大于等于 1440px：站点卡片可 4 列；
- 表格详情列在移动端转换为语义键值，不做横向挤压；
- 图表最小高度 280px，移动端 tooltip 使用点击锁定；
- Dialog 最大宽度适配内容，复杂长表单使用 Drawer 而不是超高 Dialog。

### 41.2 无障碍

- 所有图标按钮有 aria-label；
- 状态更新区域需要时使用 aria-live=polite，但 60 秒轮询不重复播报未变化内容；
- 表单 Label 与输入关联，错误通过 FieldError；
- AlertDialog 初始焦点落取消按钮；
- 数据表排序头暴露 aria-sort；
- 键盘可到达卡片操作、Tabs、筛选、分页和图表替代数据表；
- 图表必须提供“查看数据表”切换，不能作为唯一信息载体；
- 触摸目标不小于 40px；
- Skeleton 不进入可访问树或标记 aria-hidden。

## 42. 导出前端闭环

点击导出：

1. 打开 ExportDialog；
2. 显示当前统计类型、时间范围、筛选和完整性；
3. 选择 Excel 或 CSV；
4. partial/missing 时显示醒目确认，但允许导出；
5. POST 创建任务；
6. 打开 ExportJobs Sheet，2 秒轮询；
7. success 后显示文件名、行数、大小、过期时间和下载；
8. failed 显示安全错误并提供“按相同条件重新导出”；
9. expired 显示“文件已过期”，提供重新导出。

全局 Header 和全部统计页都提供“导出任务”入口，跳转 `/exports`；因此离开原统计页、刷新或重新登录后仍能找回当前用户任务。列表支持状态筛选，exportId 深链打开详情；“按相同条件重新导出”直接使用强类型 ExportFilters，不从显示文案或当前 DOM 反推参数。

导出按钮不会直接把当前表格 DOM 转文件。ExportDialog 明确说明：

- 导出不受当前分页限制；
- 包含原始 quota；

## 44. 国际化、性能和构建约束

### 44.1 国际化
