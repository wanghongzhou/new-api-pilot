## 11. 站点与通道

### 11.1 site

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint PK | 平台站点 ID |
| name | varchar(128) | 站点名称 |
| base_url | varchar(255) binary unique | 规范化 API 地址：禁止 query/userinfo/fragment，scheme/host 小写、移除默认端口和尾斜杠，路径保留大小写 |
| config_version | int | 站点地址、授权/授权状态、required capability 可用性或 enable/disable 生命周期变化时递增；远端任务写入前必须核对的统一栅栏版本 |
| remark | varchar(500) | 备注 |
| management_status | varchar(16) | active/disabled |
| online_status | varchar(16) | unknown/online/offline |
| auth_status | varchar(16) | unauthorized/authorized/expired |
| statistics_status | varchar(32) | pending_config/backfilling/ready/partial/error/paused |
| health_status | varchar(16) | ok/warning/critical/unavailable |
| root_user_id | bigint nullable | 授权后保存站点首个且不可删除的 root 用户 ID，不得指向后建 root/admin |
| root_created_at | bigint nullable | 首次授权时读取的 root 用户原始注册时间，用于重新授权校验 |
| access_token_encrypted | mediumtext nullable | 加密后的 Access Token；未授权时为空 |
| version | varchar(64) | new-api 版本 |
| system_name | varchar(128) | 站点系统名 |
| quota_per_unit | decimal(30,10) nullable | 当前 quota 换算配置 |
| usd_exchange_rate | decimal(30,10) nullable | 当前美元兑人民币配置 |
| last_rate_at | bigint nullable | 最近成功同步站点汇率时间 |
| data_export_enabled | tinyint | 是否开启数据导出 |
| current_rpm | bigint | 当前 RPM |
| current_tpm | bigint | 当前 TPM |
| last_realtime_stat_at | bigint nullable | RPM/TPM 最近成功时间 |
| probe_fail_count | int | 连续探活失败数 |
| last_probe_at | bigint nullable | 最近探活时间 |
| last_probe_success_at | bigint nullable | 最近探活成功时间 |
| statistics_start_at | bigint nullable | 授权后由 root.created_at 向下取北京时间整点得到的不可变历史起点 |
| statistics_start_source | varchar(16) nullable | 未授权为 null；首次授权后固定 root_created_at |
| statistics_end_at | bigint nullable | 永久下线时的统计终止时间 |
| disabled_at | bigint nullable | 临时停用起点；恢复回填全部成功后清空 |
| monitoring_start_at | bigint nullable | 首次通过资源能力检查时的分钟边界；资源历史不承诺该时间之前的数据 |
| created_at | bigint | 创建时间 |
| updated_at | bigint | 更新时间 |

站点状态必须独立展示，不能用单字段覆盖在线、授权、统计和资源状态。

site.name 允许重复，唯一身份是 site.id；接入去重只使用规范化后的 base_url。

`site_capability` 按 `(site_id, capability_key)` 唯一保存 §30.11 每项最近 status、message_code、message_params、checked_at。它是站点详情和重检恢复的持久依据，不保存密码、Token 或上游完整响应。

### 11.2 站点地址修改

- base_url 允许修改；
- 沿用原 site_id 和历史数据；
- 校验 URL 和唯一性；
- 修改期间暂停该站点全部任务；
- 先通过候选地址预检接口执行不携带 Token 的公开探活，并展示新旧主机、系统名和版本差异；提交修改时必须携带未过期且绑定当前 config_version 的预检凭证和同站确认；
- base_url 的主机、端口或协议发生变化时，auth_status 立即变为 unauthorized，禁止向新地址发送旧 Token；
- admin 必须重新授权并确认新地址仍代表同一逻辑站点，之后才能恢复采集；
- 仅路径变化且主机、端口、协议不变时，才允许在明确确认后继续使用原 Token；
- 无法确认是同一逻辑站点时必须新建 site，禁止把另一站点事实写入原 site_id。
- 地址或授权事务使用 §31 的 BumpSiteFence：递增 config_version 并原子终止旧父子任务、清 active_key；所有已领取任务在写库前锁定 site 并核对冻结版本，旧版本响应直接丢弃并以 SITE_CONFIG_CHANGED 结束，不能写入事实、当前状态或元数据。

### 11.3 site_channel

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint PK | 平台通道 ID |
| site_id | bigint | 所属站点 |
| remote_channel_id | bigint | new-api 通道 ID |
| name | varchar(255) | 通道名称 |
| last_synced_at | bigint | 最近同步时间 |
| remote_missing | tinyint | 远端是否已不存在 |

唯一键：

~~~text
(site_id, remote_channel_id)
~~~

`site_channel` 负责 usage 维度的稳定名称关联；渠道当前健康、余额、响应时间、missing 生命周期和告警指标由 `site_channel_inventory` 及其小时历史负责，两类模型在完整渠道快照事务内同步安全身份字段。

## 12. 客户与账户

### 12.1 customer

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint PK | 客户 ID |
| name | varchar(128) | 客户名称 |
| contact | varchar(255) | 联系方式 |
| remark | varchar(500) | 备注 |
| status | varchar(32) | communicating/signing/using/disabled |
| statistics_paused_at | bigint nullable | 停用统计开始时间 |
| statistics_backfill_status | varchar(16) | none/pending/running/failed |
| created_at | bigint | 创建时间 |
| updated_at | bigint | 更新时间 |

客户与账户是一对多关系，直接通过 account.customer_id 表达，不建立 customer_account 中间表。

customer.name 允许重复，页面和导出必须同时携带 customer.id，不得用名称作为关联键。

### 12.2 account

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint PK | 平台账户 ID |
| site_id | bigint | 所属站点，不可修改 |
| customer_id | bigint | 所属客户，不可修改 |
| remote_user_id | bigint | new-api user ID，不可修改 |
| remote_created_at | bigint | 远端用户注册时间快照，不可修改；账户统计历史起点 |
| username | varchar(255) | 当前远端用户名 |
| display_name | varchar(255) | 当前远端显示名 |
| remote_group | varchar(128) | 当前远端分组 |
| remote_status | int | 当前远端状态 |
| remote_state | varchar(32) | normal/missing/identity_mismatch |
| remote_missing_count | int | 连续完整用户快照未出现次数 |
| last_remote_seen_at | bigint nullable | 最近一次在完整远端用户快照中出现的时间 |
| quota | bigint | 当前剩余额度快照 |
| used_quota | bigint | 当前累计已用快照 |
| request_count | bigint | 当前累计请求快照 |
| managed_status | varchar(16) | active/archived |
| statistics_paused_at | bigint nullable | 账户统计有效暂停起点，来源为账户归档或所属客户停用 |
| statistics_backfill_status | varchar(16) | none/pending/running/failed |
| last_synced_at | bigint | 最近属性同步时间 |

| created_at | bigint | 纳管时间 |

唯一键：

~~~text

~~~

site_id、customer_id、remote_user_id、remote_created_at 创建后均不可修改。

### 12.3 远端账户变化

- 用户名变化：同步 account.username；
- 远端接口后续返回的 created_at 与已存 remote_created_at 不一致时不自动覆盖，remote_state=identity_mismatch，并从当前整点写入 statistics_paused_at，防止复用 ID 的新用户用量进入原客户；
- 显示名、分组、额度变化：更新当前快照；
- 远端 status=1 视为启用；其他值视为非启用，更新快照并触发账户禁用告警；
- 远端查询不到：remote_state=missing，触发告警；
- 仅完整用户快照参与缺失计数；连续两次未出现才变为 missing，任一次重新出现立即清零 remote_missing_count、更新 last_remote_seen_at 并恢复 normal；
- 不自动删除或归档；
- 后续重新出现时恢复 normal。
- identity_mismatch 在本期不可自动或手工恢复原绑定，因为冲突期间的事实身份无法安全判定。admin 只能归档旧账户，或在完全无历史时删除；若要继续纳管复用 ID，属于本期排除的历史归属变更，必须另行设计，不能把新身份接到旧账户。

### 12.4 账户归档与恢复

- 归档期间暂停账户和所属客户的新增汇总；
- 基础事实、站点、全局、模型、通道统计继续生成；
- 恢复后从基础事实补齐整个归档期间；
- 补齐完成后账户和客户历史与未归档时一致。

### 12.5 客户停用与恢复

- disabled 期间暂停客户及其账户汇总；
- 停用事务将客户 statistics_paused_at 写入其全部 active 账户的 statistics_paused_at（仅在为空或更晚时取更早值），但不改变账户 managed_status；
- 基础事实、站点、全局、模型、通道统计继续生成；
- 恢复为 using 后补齐整个停用期间，但 identity_mismatch 账户保持暂停且不参与补齐；
- 客户停用不会改变账户自身 managed_status；
- 本身 archived 的账户仍需单独恢复。

### 12.6 删除规则

- account 没有任何统计数据时允许物理删除；
- customer 没有账户和统计数据时允许物理删除；
- site 没有账户和统计数据时允许物理删除；
- 已有历史数据时禁止删除，只能停用或归档；
- 存在 pending/running 任务或 firing 告警时禁止物理删除；
- 不使用 deleted_at；
- 不使用 ON DELETE CASCADE。

### 12.7 暂停期间的汇总边界

- account 为 archived 时，账户和客户汇总只计算 statistics_paused_at 之前的数据；
- 账户在 `floorToBeijingHour(remote_created_at)` 之前不属于 expected 范围，完整零值和缺口均不计入账户/客户完整率；
- customer 为 disabled 时，该客户及其账户汇总只计算 statistics_paused_at 之前的数据；
- 站点、全局、模型、通道汇总不受账户或客户暂停影响；
- 恢复时先保留 statistics_paused_at，完成对应区间补算后再清空；identity_mismatch 账户永不由普通恢复流程清空；
- 补算失败时保留暂停时间和失败状态，避免把未完成数据标记为完整；
- 任意历史重建任务都必须遵守当前暂停边界；
- 历史重建按目标小时与 statistics_paused_at 比较，不得用当前 managed_status/customer.status 直接过滤全部历史：目标小时早于暂停边界的既有统计始终保留，边界及之后才是 paused；
- 恢复并补算完成后，历史结果与从未暂停时一致。

## 14. 实例资源统计

### 14.1 采集原则

- 每 60 秒请求一次 /api/system-info/instances；
- 按实例保存分钟数据；
- 不做主机 hostname 去重；
- 站点最大值和平均值直接按实例聚合；
- 告警基于分钟数据；
- RPM/TPM 不进入资源时间序列。
- 资源历史只从 monitoring_start_at 开始；站点主动停用区间记为 paused 且不回填，接口失败/离线/授权异常造成的应采未采记为 missing。

### 14.2 资源监控区间

`site_monitoring_pause` 保存 intentional pause：site_id、start_minute_ts、end_minute_ts、reason、created_at，唯一键 `(site_id,start_minute_ts)`。站点 disable 时插入 end=null 的区间，enable 时在开始用量回填的同时把 end 关闭为当前分钟；资源采集从该分钟恢复，不等待历史用量回填完成。永久下线保持区间关闭边界并停止新增 expected 分钟。

资源 expected 分钟满足：minute>=monitoring_start_at、未落入 pause 区间、早于 statistics_end_at（如有）。历史查询通过该表区分 paused 与 missing；不能在恢复后清空停用证据。

### 14.3 site_instance

保存实例当前信息：

- site_id；
- node_name；
- hostname；
- is_master；
- runtime_version；
- goos；
- goarch；
- upstream_status、upstream_stale_after_seconds；
- current_status：按实际生效 `instance_stale` 规则计算的 online/stale/offline/unknown；
- first_seen_at；
- started_at；
- last_seen_at；
- last_synced_at。
- retired_at：nullable Unix 秒；仅成功完整实例快照确认节点消失时写入，重新出现时清除；失败、非法或 unknown 快照不得修改。

唯一键：

~~~text
(site_id, node_name)
~~~

node_name 与上游原值按字节区分大小写，相关实例表统一使用 `utf8mb4_bin`，不按 hostname 合并。

`site_instance` 只表示当前目录，不足以表达 retire→gap→reappear。`site_instance_lifecycle` 永久保存节点活动区间：`site_id/node_name/start_minute_ts/end_minute_ts/evidence_status/created_at/updated_at`；起止均为分钟边界，区间采用 `[start,end)`，`end_minute_ts=NULL` 表示当前开放。唯一键 `(site_id,node_name,start_minute_ts)` 保证重放，同一 `(site_id,node_name)` 最多一个开放区间；同一分钟 retire 后 reappear 合并为开放区间，不产生零长重复段。

实例 expected 只从 `evidence_status=known` 的生命周期区间计算，并与站点 monitoring_start/statistics_end/pause 取交集。首次出现从 first_seen_at 向下取整的分钟开始；成功完整快照确认消失时在退休时刻所在分钟关闭区间，失败、非法或 unknown 快照不得关闭；重新出现从新样本分钟打开新区间。迁移 0020 从旧 current 行回填的区间标记 `legacy_unknown`，仅用于保守迁移诊断，不得据此伪造历史 expected、重写 complete/final 桶或推进 finalize 水位；首次新成功样本在该分钟切断 legacy 区间并建立 known 区间。

D141 的 resource scope revision 是目标北京时间日期内以下元组规范化后的 SHA-256 小写十六进制：所有候选 site 的 `(site_id,config_version,monitoring_start_at,statistics_end_at)`、相交 pause 的 `(id,site_id,start_minute_ts,end_minute_ts,created_at)`，以及相交 known lifecycle 的 `(id,site_id,node_name,start_minute_ts,end_minute_ts,updated_at)`。唯一编码为无空白 UTF-8 JSON 数组 `[["sites",rows],["pauses",rows],["lifecycles",rows]]`；section 顺序固定如前，row 按各自完整元组升序，整数为 JSON 十进制数、nullable 为 JSON `null`、字符串使用标准 JSON escaping，不允许 map/object 或实现自定分隔符。Go、SQL 辅助查询和 A102 必须复用同一 canonical helper/fixture vectors。`data_maintenance_state.scope_revision` 保存全局扫描 revision；每个 item 候选另携带对应 site slice revision。提交短事务必须先锁 site，再锁相交 pause/lifecycle，重算 site revision 后才写桶和推进复合游标。扫描完成前重算全局 revision；不一致则清空复合游标并保持 pending，不能 complete/final。已结束前一日也可能被补录 pause/lifecycle 证据改变，因此不得假定当前快照天然不可变。

### 14.6 留存

- 实例和站点 minutely 默认保留 90 天；
- 实例和站点 hourly/daily 永久保留；
- 业务用量 hourly/daily 永久保留；
- 清理分钟数据前必须确认小时、天汇总已完成；
- 分钟留存天数可配置。

# D139 system-task 只读监控模型

`site_system_task` 保存安全投影：`id BIGINT`、`site_id BIGINT`、`remote_id BIGINT`、`task_id VARCHAR(64)`、`type VARCHAR(32)`、`status VARCHAR(16)`、`created_at BIGINT`、`updated_at BIGINT`、nullable typed progress 计数与百分比、按五类 type 展开的 nullable typed result 计数、`error_present BOOLEAN`、nullable `error_code VARCHAR(64)`、`first_seen_at/last_seen_at/collected_at BIGINT` 和 `site_config_version BIGINT`。唯一键为 `(site_id,remote_id)`，另建 `(site_id,task_id)` 唯一键；所有计数使用 int64，对 JSON 输出为十进制字符串。type/status/error_code 使用 D139 固定枚举 CHECK；progress 为 0..100。

表中禁止 `active_key`、`locked_by`、payload/state/result JSON、error 原文或通用 extra/raw JSON 列。typed result 的非本 type 字段必须为 NULL，Service 在写入前验证判别联合；未知字段只在 decoder 丢弃，不能通过 JSON 逃生舱落库。

`site_system_task_collection_state` 以 `(site_id,resource_key)` 唯一，`resource_key` 只允许 `list` 与 `current:<五类 type>`，保存 `data_status`、`last_attempt_at`、`last_success_at`、`last_failure_at`、`as_of`、`site_config_version`、安全 `failure_code`、`list_limit=100`、`truncated`、`id_gap` 和最近安全水位。它是重启恢复、同步 failure 和 config fence 的 durable 事实，不以进程内游标替代。terminal retention 只删除 `succeeded/failed` 且 `updated_at` 早于平台截止点的行；`pending/running` 不受年龄清理。
